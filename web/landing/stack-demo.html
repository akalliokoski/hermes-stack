Hermes Agent + hermes-stack demo

Self-hosted AI operator stack for chat, memory, automation, and media workflows

Hermes runs natively on the VPS as the operator. This repo, hermes-stack, supplies the environment rendering, profile provisioning, service wiring, docs, and deployment conventions around it. Docker Compose handles the support layer, while Tailscale exposes selected UIs to the tailnet without making the services public.

Architecture map

The most important split is: Hermes itself stays native and close to upstream, while the surrounding services run in Docker Compose and are selectively published through Tailscale.

Service tour

Hermes Dashboard

Built-in Hermes web UI for configuration, sessions, logs, and credentials. Runs as a host-native service and is exposed through a dedicated Tailnet HTTPS port.

local 127.0.0.1:9119 → tailnet :9444

Firecrawl

Gives Hermes a crawling and extraction backend for richer web research than basic fetch/search alone.

127.0.0.1:3002 and /firecrawl/

Hindsight

Provides semantic long-term memory. Profiles share the service instance but stay isolated by bank IDs like hermes-default and hermes-gemma.

127.0.0.1:8888 API, 127.0.0.1:9999 UI, tailnet :9443

Litestream + backups

Protects Hermes state with continuous SQLite WAL replication plus scheduled snapshot archives and logical Hindsight SQL dumps.

Syncthing + shared wiki

Syncs /home/hermes/sync across devices, including the shared wiki and synchronized backup destinations.

How the stack is operated

The repo is the control plane. Hermes is the operator. Direct shell debugging exists, but the normal path is to ship changes through version control and let deploy automation apply them.

1

Render environment and profiles

config/base.yaml and environment overlays render profile config, service URLs, environment context, and shared SOUL composition.

2

Provision profile behavior

scripts/provision-profile.sh creates workspaces, Hindsight bank wiring, profile SOUL output, and optional gateway units.

3

Run support services

Compose keeps Firecrawl, Hindsight, durability helpers, sync, and media apps available on localhost.

4

Publish safely to the tailnet

Tailscale Serve exposes only the selected UIs, keeping the rest of the stack private and localhost-bound.
